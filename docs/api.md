# API Reference - Справочник API

Базовый адрес локального запуска:

```text
http://127.0.0.1:8080
```

Все ответы сервиса возвращаются в JSON.

Формат ошибки:

- `{"error":"..."}`

## Общие коды ответа

- `200` — успешный запрос
- `201` — объект создан
- `400` — неверные параметры или JSON
- `401` — ошибка аутентификации
- `403` — нет доступа
- `404` — сущность не найдена
- `409` — конфликт данных
- `500` — внутренняя ошибка сервера или БД

## Auth - Аутентификация

### `POST /api/register`

Регистрирует нового пользователя.

Тело запроса:

- `login` — уникальный логин
- `password` — пароль (от 8 до 32 символов, минимум: 1 заглавная, 1 строчная, 1 цифра, без пробелов)
- `name` — отображаемое имя

Возможные коды ответа:

- `201` — пользователь создан
- `400` — невалидные поля или JSON
- `409` — логин уже занят
- `500` — ошибка БД

### `POST /api/login`

Выполняет вход.

Тело запроса:

- `login` — логин
- `password` — пароль

Возможные коды ответа:

- `200` — успешный вход
- `400` — невалидные поля или JSON
- `401` — неверный логин или пароль

## Users - Пользователи

### `GET /api/user/profile`

Возвращает профиль пользователя.

Параметры строки запроса:

- `user_id` — обязательный положительный `int`

Ответ содержит:

- `user_id`
- `login`
- `name`
- `bio`

Возможные коды ответа:

- `200` — профиль получен
- `400` — невалидный `user_id`
- `404` — пользователь не найден
- `500` — ошибка БД

### `PATCH /api/user/profile`

Обновляет профиль пользователя или пароль.

Тело запроса:

- `user_id` — обязательный положительный `int`
- `name` — необязательная строка
- `bio` — необязательная строка
- `login` — необязательная строка
- `new_password` — необязательная строка (те же требования, что при регистрации)
- `current_password` — обязателен, если меняется пароль

Особенности:

- должен быть передан хотя бы один обновляемый параметр: `name`, `bio`, `login`, `new_password`
- если меняется пароль, сервер проверяет текущий пароль

Возможные коды ответа:

- `200` — профиль обновлен
- `400` — невалидные поля или JSON
- `403` — неверный текущий пароль
- `404` — пользователь не найден
- `409` — новый логин уже занят
- `500` — ошибка БД

### `GET /api/users/search`

Поиск пользователей.

Параметры строки запроса:

- `query` — обязательная строка, минимум 2 символа

Особенность:

- поиск выполняется по полю `name` (ILIKE), результат ограничен 10 записями

Возможные коды ответа:

- `200` — поиск выполнен
- `400` — пустой запрос или строка меньше 2 символов
- `500` — ошибка БД

## Chats - Чаты

### `GET /api/chats`

Возвращает список чатов пользователя.

Параметры строки запроса:

- `user_id` — обязательный положительный `int`
- `limit` — необязательный, по умолчанию `50`, диапазон `[1..200]`
- `offset` — необязательный, по умолчанию `0`, неотрицательный `int`

Возможные коды ответа:

- `200` — список чатов получен
- `400` — невалидные параметры запроса
- `404` — пользователь не найден
- `500` — ошибка БД

### `POST /api/chats/group`

Создаёт групповой чат.

Тело запроса:

- `name` — название чата
- `creator_id` — положительный `int`
- `participants` — непустой массив положительных `user_id`

Особенности:

- создатель автоматически добавляется в участников, если его нет в массиве
- дубликаты в `participants` удаляются
- все указанные пользователи должны существовать

Возможные коды ответа:

- `201` — групповой чат создан
- `400` — невалидные поля или JSON
- `500` — ошибка БД

### `GET /api/chat/{chat_id}`

Возвращает информацию о чате и список участников.

Параметры пути:

- `chat_id` — обязательный положительный `int`

Параметры строки запроса:

- `user_id` — необязательный положительный `int`

Если `user_id` передан, сервер проверяет, что пользователь состоит в чате.

Ответ содержит:

- `chat_id`
- `name`
- `type_name` — `group` или `private`
- `participants` — массив объектов `{ user_id, name }`
- `participants_count`

Возможные коды ответа:

- `200` — информация о чате получена
- `400` — невалидные `chat_id` или `user_id`
- `403` — пользователь не участник чата
- `404` — чат не найден
- `500` — ошибка БД

### `PATCH /api/chat/{chat_id}`

Переименовывает групповой чат.

Тело запроса:

- `user_id` — ID пользователя, который делает запрос
- `name` — новое название чата

Ограничения:

- доступно только для групповых чатов
- пользователь должен быть участником чата

Возможные коды ответа:

- `200` — название чата обновлено
- `400` — невалидные данные или чат не групповой
- `403` — пользователь не участник чата
- `404` — чат не найден
- `500` — ошибка БД

### `POST /api/chat/{chat_id}/users/add`

Добавляет участника в групповой чат.

Тело запроса:

- `requester_id` — ID пользователя, который делает запрос
- `user_id` — ID добавляемого пользователя

Возможные коды ответа:

- `200` — пользователь добавлен
- `400` — невалидные данные или чат не групповой
- `403` — requester не участник чата
- `404` — чат или пользователь не найден
- `409` — пользователь уже в чате
- `500` — ошибка БД

### `POST /api/chat/{chat_id}/users/remove`

Удаляет участника из группового чата.

Тело запроса:

- `requester_id` — ID пользователя, который делает запрос
- `user_id` — ID удаляемого пользователя

Возможные коды ответа:

- `200` — пользователь удален из чата
- `400` — невалидные данные, чат не групповой, либо попытка удалить себя
- `403` — requester не участник чата
- `404` — чат не найден или пользователь не состоит в чате
- `500` — ошибка БД

### `POST /api/chat/{chat_id}/leave`

Пользователь выходит из чата.

Тело запроса:

- `user_id` — ID пользователя, который выходит из чата

Возможные коды ответа:

- `200` — пользователь вышел из чата
- `400` — невалидные данные или чат приватный
- `404` — чат не найден или пользователь не состоит в чате
- `500` — ошибка БД

## Messages - Сообщения

### `POST /api/messages/send`

Отправляет сообщение.

Тело запроса:

- `sender_id` — положительный `int`
- `content` — непустой текст
- `chat_id` — неотрицательный `int`
- `receiver_id` — обязателен для личного чата (`chat_id = 0`)

Особенности:

- если `chat_id = 0`, сервис создаёт или использует личный чат между `sender_id` и `receiver_id`
- при `chat_id = 0` поле `receiver_id` должно быть положительным
- при `chat_id = 0` нельзя отправлять сообщение самому себе (`sender_id != receiver_id`)
- если `chat_id > 0`, отправитель должен быть участником чата

Возможные коды ответа:

- `201` — сообщение отправлено
- `400` — невалидные данные или JSON
- `403` — отправитель не участник чата (в том числе при несуществующем `chat_id > 0`)
- `404` — отправитель или получатель не найден
- `500` — ошибка БД

### `GET /api/messages`

Возвращает список сообщений чата.

Параметры строки запроса:

- `chat_id` — обязательный неотрицательный `int`
- `user_id` — обязательный положительный `int`
- `limit` — необязательный, по умолчанию `50`, диапазон `[1..200]`
- `offset` — необязательный, по умолчанию `0`, неотрицательный `int`

Возможные коды ответа:

- `200` — сообщения получены
- `400` — невалидные параметры запроса
- `403` — пользователь не участник чата

### `PATCH /api/messages/{message_id}`

Редактирует сообщение.

Тело запроса:

- `user_id` — ID автора сообщения
- `content` — новый непустой текст

Ограничение:

- редактировать может только автор сообщения

Возможные коды ответа:

- `200` — сообщение отредактировано
- `400` — невалидные данные или JSON
- `403` — пользователь не автор сообщения
- `404` — сообщение не найдено
- `500` — ошибка БД
